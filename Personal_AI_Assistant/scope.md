# Scope — Personal AI Assistant

本文件定義這個 Personal AI Assistant 的使用範圍與安全規則。
分為兩部分：**工具本身內建的機制**，以及**我額外要求 AI 遵守的規則**。
兩者性質不同，請勿混淆。

## 一、工具本身內建的機制（Claude Code 原本就有）

- **權限模式**：可設定不同的確認頻率（例如每個操作都要確認、或只在高風險操作時確認）。這是唯一目前實際存在、由使用者掌控的一層。
- **執行前提示**：對於會影響共享狀態或難以復原的操作，Claude Code 的預設行為是先說明將要做什麼、再等待確認，而不是自動拒絕或自動執行。
- 目前**沒有**自動「垃圾桶」機制（刪除不會自動變成可還原狀態），也**沒有**寫死的指令黑名單會讓系統自動拒絕執行。

## 二、我（使用者）額外要求 AI 遵守的規則

這些是本專案自訂的安全規則，AI 應在對話中主動遵守：

1. **不要求提供機密資料**
   AI 不得要求我貼上密碼、API key、私人資料、薪酬資料、員工紀錄或客戶紀錄。

2. **高風險指令一律先問，不自動執行**
   涉及以下性質的操作，AI 必須先用白話解釋「這個指令會做什麼」，並等待我明確確認後才執行：
   - 刪除檔案／資料夾（包含看似「清理」的操作）
   - 使用 `force`、`--hard`、`--force` 等強制參數
   - 需要 admin／系統管理員權限的操作
   - 任何涉及密碼、金鑰、憑證（credential）的操作

3. **預設保守，寧可多問一次**
   若不確定某個操作是否可逆、是否會影響此資料夾以外的範圍，AI 應先詢問，而不是自行判斷後執行。

4. **Session 記錄不寫入敏感資料**
   `sessions/` 資料夾內的紀錄，不得包含密碼、API key、私人資料、薪酬資料、員工紀錄或客戶紀錄（詳見 `sessions/README.md`）。

## 三、範圍界定

- 這個 Personal AI Assistant 目前的用途：`AI_Agent_Training` 培訓期間的學習與實驗紀錄。
- 不涉及正式生產環境、真實客戶資料或公司內部系統。
